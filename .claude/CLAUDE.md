# 基本的なルール

- プロダクトコードを作成・変更した場合は、次のような確認を行い問題が発生していないことを確認する。テストコード、ドキュメントは除く
  - ファイルの修正後に、`pnpm prettier`、`pnpm lint --fix` でファイルのフォーマットを行う
  - `pnpm test:run` を実行し、修正したファイルを個別にテスト実行する。テストケースを修正すべきか判断に迷った場合は確認する
  - `pnpm lint` を実行し、修正したファイルを個別に lint チェックを行う。問題があれば修正する
  - `pnpm tsc` を実行し、TypeScriptのチェックを行う。問題があれば修正する
- テストコードの作成・変更をする場合はスキルを使用する
- コミットをする場合はスキルを使用する
- .ts、.tsx のファイルには適切な TSDoc を記述する
